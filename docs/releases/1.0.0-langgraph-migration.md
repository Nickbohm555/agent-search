# agent-search 1.0.0 LangGraph Migration Release Notes

`agent-search-core` `1.0.0` is the SemVer-major release that formalizes the LangGraph-native runtime as the supported architecture for in-process SDK usage and repository documentation.

## Release Scope

- Declares `1.0.0` as the stable major-version target for the LangGraph runtime and typed state-graph contracts.
- Treats legacy orchestration behavior as compatibility-only surface area rather than the primary programming model.
- Gates publication on documentation parity, OpenAPI/reference synchronization, and runnable SDK verification.

## Highlights

- The supported SDK-first entrypoint is `advanced_rag(...)` for synchronous execution with LangGraph-native runtime state handling.
- Async and status APIs remain available for durable execution flows aligned with the LangGraph runtime lifecycle.
- State, reducer, and node I/O contracts are now documented as release-era compatibility boundaries instead of internal implementation details.

## Breaking Changes

- `1.0.0` is a SemVer-major release because the LangGraph runtime is the canonical architecture and migration expectations are explicit for existing integrators.
- `run(...)` is retained only as a deprecated compatibility alias; new integrations should migrate to `advanced_rag(...)`.
- `langfuse_settings` is deprecated for tracing setup. Pass an explicit `langfuse_callback` instead of relying on compatibility behavior.
- Migration-sensitive documentation now assumes LangGraph-first terminology, runtime flow, and verification steps.

## Migration Prerequisites

Before adopting `1.0.0`, confirm the following:

- Python version is within the supported `>=3.11,<3.14` range.
- Your integration can supply both a compatible chat model and a vector store implementing `similarity_search(query, k, filter=None)`.
- Your deployment or local environment can validate the current API/OpenAPI contract and SDK examples before release rollout.
- Your team has reviewed the [Migration Guide](../migration-guide.md) and [Deprecation Map](../deprecation-map.md) for stepwise changes and removal horizons.

## release-candidate checklist

The release-candidate checklist must be complete before publishing `agent-search-core` `1.0.0`:

- Confirm `sdk/core/pyproject.toml` declares `version = "1.0.0"`.
- Build the SDK package with `cd sdk/core && python -m build`.
- Run `./scripts/release_sdk.sh` without `PUBLISH=1` and confirm the built wheel contains the `agent_search` package.
- Validate release documentation parity, including the [Migration Guide](../migration-guide.md) and [Deprecation Map](../deprecation-map.md).
- Validate OpenAPI/reference parity with `uv run --project src/backend python scripts/export_openapi.py --output openapi.json` and `./scripts/validate_openapi.sh`.
- Run the baseline example `python sdk/examples/run_health.py --base-url http://localhost:8000`.
- Confirm the root repository README links to these release notes and migration materials for immediate discoverability.

## Support Policy For 1.0.0

- LangGraph-first runtime behavior is the supported path for new work.
- Deprecated compatibility shims remain documented until their removal horizon is published in the deprecation map.
- Any future removal of compatibility aliases should be announced in release notes and reflected in migration/deprecation documentation before publication.
