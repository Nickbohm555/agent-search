# agent-search-core 1.0.14

`agent-search-core` `1.0.14` exposes per-call runtime step toggles and tightens prompt customization so caller overrides cannot replace the runtime-injected question and evidence inputs.

## What changed

- `advanced_rag(...)` now accepts `rerank_enabled=` and `query_expansion_enabled=` keyword arguments.
- These explicit kwargs override any matching values under `config["runtime_config"]` for that call.
- The in-process runtime now honors those toggles during expansion and rerank execution.
- `custom_prompts.subanswer` and `custom_prompts.synthesis` now override only the instruction block.
- The SDK always appends `sub_question`, `main_question`, and evidence/context sections after the custom instructions.

## Behavior notes

- Disabling query expansion bypasses the expand node and uses the original sub-question as the search query.
- Disabling rerank bypasses the rerank node and forwards retrieved rows in their original order.
- Prompt overrides can still change tone, structure, and policy instructions, but not the placement or substitution of runtime inputs.

## Example

```python
from langchain_openai import ChatOpenAI

from agent_search import advanced_rag
from agent_search.vectorstore.langchain_adapter import LangChainVectorStoreAdapter

vector_store = LangChainVectorStoreAdapter(your_langchain_vector_store)
model = ChatOpenAI(model="gpt-4.1-mini", temperature=0.0)

response = advanced_rag(
    "What changed in NATO maritime policy?",
    vector_store=vector_store,
    model=model,
    rerank_enabled=False,
    query_expansion_enabled=True,
    config={
        "custom_prompts": {
            "subanswer": "Answer with concise cited evidence only.",
            "synthesis": "Write a short synthesis and preserve citation markers.",
        }
    },
)
```
